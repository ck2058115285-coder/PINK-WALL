## 一、 数据库迁移与环境恢复
* **旧站环境恢复**：成功提取并整合了旧站 `uploads` 媒体库、自定义主题与 `Access`、`Meting` 等基础插件，补齐了遗失在根目录的 `photo` 独立页面静态资源。
* **MySQL ➔ SQLite 转换**：
  * 解析了旧站 MySQL 导出的 `localhost.sql` 数据文件。
  * 编写 Python 迁移脚本自动解决保留字冲突（如 `` `order` ``、`` `group` `` 字段转义）及表结构缺失（如为 SQLite 的 `typecho_contents` 表自动补足旧插件的 `views` 浏览量字段）。
  * 成功转换并写入本地 `2026.db` SQLite 数据库，完整恢复了 3 名用户、45 篇文章与 4 万多条历史访问日志。
* **绝对域名清理**：全表扫描并移除了文章正文及附件中历史失效的 `gosoloto.top` 绝对域名，使其退化为本地相对路径，解决图片 404 挂机问题。

## 二、 首页“无分页”无限平铺优化
* **不分页模式**：在 `functions.php` 的 `themeInit` 周期中挂载逻辑，当检测到首页访问时动态将查询的 `pageSize` 重写为 `9999`，实现所有文章的一页平铺，创造沉浸式相册观感。
* **多语翻译修正**：将首页到底提示 “别滑了，没有了” 及导航栏首个按钮 “首页” 处的原生翻译函数由仅返回值的 `_t()` 变更为带自动打印功能的 `_e()`，修复了之前渲染出空容器的 Bug。

## 五、 借鉴 极简卡片式大图 风格的视觉升级（减法设计）
* **功能极致精简**：
  * **砍掉 Explore**：删除了导航栏 “发现” 按钮、首页 Explore 磁贴卡片、以及页面底部的抽屉侧栏相关的所有冗余 JS 交互、HTML 结构与 CSS 样式。
  * **砍掉二十四节气**：删除了原版磁贴与 `functions.php` 中耗费性能的公历农历二十四节气推算代码。
  * **砍掉点赞分享**：剥离了卡片右上角的心形 Likes 统计及 clipboard 链接分享功能，删除了对应的 AJAX 处理路由与提示 Toast，使首页 DOM 节点减少了 40%，首屏极速加载。
* **巨幕铺满与融合过渡**：
  * **16:9 比例限高**：通过 CSS 的 `aspect-ratio: 16 / 9` 强制轮播大图在桌面端保持完美横宽比，并将最大高度限制为 `70vh`，防止在大屏或矮视口设备上出现小圆点和滚动下翻按钮被推下首屏的情况。
  * **顶栏透明重合**：使幻灯片巨幕向上负移 `70px`，铺满至视口最顶端；顶栏（Logo 与导航栏）背景不透明度设为 `0.1`，辅以 `20px` 深度毛玻璃模糊，实现幻灯片画面穿透融合的玻璃光感。
  * **底部渐隐过渡**：为幻灯片层添加了 `120px` 高度从 `transparent` 渐隐到 `var(--bg-color)` 页面背景色的线性蒙版，实现大图与下方文章流之间的平滑视觉过渡。
* **图集封面单图化**：
  * 瀑布流中的多图文章卡片仅展示第一张封面图，取消原先的多格拼图。
  * 将其余图集照片以隐藏链接形式渲染在 DOM 中，大图浏览器 (Lightbox) 在激活时扫描该卡片内的隐藏链接，允许用户在弹窗状态下使用左右按钮或键盘左右方向键进行无缝翻页。

## 七、 幻灯片图片随机性与前端 API 异步切片拉取
* **服务端随机打乱**：在 `functions.php` 的 API 拦截中调用 `shuffle()` 对大图库进行打乱。
* **防缓存策略**：API 返回时加入了 `Cache-Control: no-store, no-cache, must-revalidate` 等防缓存响应头，且前端 Fetch 请求时动态拼接时间戳，保证用户每次刷新都得到最新重排的图片组合。
* **随机切片 (Slice 6)**：
  * 每次 API 仅截取并返回打乱后的 **6 张** 大师名画（`array_slice($slides, 0, 6)`）。
  * 每次刷新展现不同的 6 张图组合，大大提高了新鲜感；同时**将原本加载 20 张大图的带宽开销降低了 70%**，有效优化了服务器负载与页面加载速度。
* **前端异步 Fetch**：在 `index.php` 预置精美 skeleton 加载占位框，`footer.php` 采用 JS `fetch()` 异步拉取 JSON 数据，随后使用 DOM 操作动态生成幻灯片和 dots 控制点，并无缝绑定轮播与手动箭头的切换。
